Recursion
함수 -> 호출관계 + 호출순서

Problem + 논리(재귀) -> Code 

호출 관계를 파악하는것은 OK.
호출 순서를 파악하는게 어렵다.
문제 자체는 복잡한데, 재귀로 접근하면 간단히 해결되는경우.

Ex) Fibonacci 피보나치 수열

피보나치 수열의 구성 
수열의 n번째 값 = 수열의 n-1번째 값 + 수열의 n-2번째 값

이진 탐색 알고리즘의 재귀 구현

1. 탐색 범위의 중앙에 목표값이 저장되었는지 확인
2. 저장되지 않았다면 탐색 범위를 반으로 줄여서 다시 탐색 시작

가장먼저 구현되는것 종료에 대한 로직(first > last)
